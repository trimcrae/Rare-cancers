(honest in-silico degrader selectivity for one target family), convert the resulting benchmarks and

## 3. Destination Architecture (ExO 3.0)

### 3.1 MTP as a three-layer machine-readable protocol (not a poster)

**MTP (candidate):** *"Make rigorous, honest in-silico drug-target evaluation abundant for
neglected diseases."*

- **Constraint Layer (categorically forbidden — hard rules the agent fleet must obey):**
  - Never state or imply efficacy, safety, therapeutic window, or clinical readiness.
  - Never present an un-benchmarked number as an accuracy claim; conditional results stay labelled conditional.
  - Never fabricate data/citations; every clinical value cited; synthetic data bannered.
  - Never spend an expensive GPU rung before its cheaper gate says GO.
  - Never ship an outward-facing artifact (preprint, client deliverable, grant) without the red-team + human sign-off.
- **Decision Layer (weighted priorities under tradeoffs):**
  1. Scientific defensibility > speed-to-publish > breadth. 2. Cheapest-decisive experiment first.
  3. A new axis of evidence (breadth) beats deepening a to-standard test. 4. Non-dilutive/founder-owned
     funding > dilutive. 5. Reusable capability > one-off result.
- **Identity Layer (who this firm is for — with explicit disqualifiers):**
  - *For:* rare/neglected-disease foundations, target-discovery/TPD teams who value honest
    negative results and reproducibility, and grant bodies funding method rigor.
  - *Disqualifiers (who it is NOT for):* anyone wanting hype metrics, an implied clinical story, a
    "selective drug" headline, or results faster than the science supports. Working with them destroys
    the moat (trust).

**Purpose Litmus Tests (all three must pass):**
1. *Could an agent, given only this MTP, make a decision leadership would endorse?* — Yes: e.g. "reframe
   'binds' → 'compatible with the hypothesized conditional bound state'" is derivable from the Constraint Layer.
2. *Could it decide what NOT to build?* — Yes: it would refuse to run the $150–400 matrix before the four
   cheaper gates say GO (execution is nearly free; gating is the discipline).
3. *Could a high-judgment human read the Identity Layer and know why they'd engage?* — Yes: honesty-as-brand.

**Agentic Fidelity Paradox guard:** encode *purpose*, not rigid procedure. The agents should reason from
the Constraint/Decision layers on novel targets, not just replay the NR4A3 SOP — GOVERN (§5) catches drift.

### 3.2 Five Design Conditions (binding Step-1 exit gate — all must hold)

1. **AI-centric workflow architecture** — ✅ met (agent-run science).
2. **Recursive-improvement infrastructure** — ✅ met (gates + method-watch re-grade).
3. **Model sovereignty & governed autonomy** — ⚠️ partial: provider-agnostic compute ✅; but **the firm's
   reasoning runs on a single foundation-model family.** Mitigation named in §5 (cognitive captivity).
4. **Intelligence density at every layer** — ✅ met by construction (N=1 running a full program).
5. **Human flourishing as a binding constraint** — ⚠️ **the founder is the "workforce"; §4 makes capacity
   a hard constraint, not an afterthought.** Condition holds only if §4 controls are adopted.

### 3.3 Three Compounding Loops (don't optimize one at another's expense)

- **Intelligence Loop (D→I→R→V):** each target campaign produces benchmarks + decision traces →
  compounds into the Value Moat. **This is the growth engine.**
- **Trust Loop (E-Trust→E-Agency→V):** honest, reproducible, cited outputs → reputation → more
  engagements/grants. **For a solo shop this loop IS the go-to-market.**
- **Governance Loop (S→A→R):** language discipline + red-team + reproducibility keeps velocity from
  producing a credibility-destroying over-claim.

**Dominant failure mode to avoid:** optimizing the Intelligence Loop (ship more predictions faster) at
the Trust Loop's expense (one over-claim). For this asset, **Trust is the scarce loop — protect it first.**

## 4. SHAPE for N=1 (the chassis — where solo firms actually die)

Standard SHAPE Human-Architecture constructs (Middle 60%, delayering, talent Binding Problem) **re-map**
for a one-person firm. Honest adaptation, not force-fit:

- **S — Safe Autonomy.** The agent fleet already touches spend (GPU $) and outward artifacts. Required:
  - **Fiduciary Wedge:** every agent action chains to the founder as the single named human owner — already true.
  - **Permission Envelope with hard thresholds:** the "$50 / expensive-spend" rule + "no outward-facing
    action without sign-off" ARE the envelope. Formalize: scope isolation (no prod-credential access for
    exploratory agents), approval threshold on any GPU rung >$50, and a soft-delete/reversibility window
    on anything published (preprint = irreversible → human gate).
  - **Kill switch:** the existing `*-stop-aws.yml` / spot-teardown wrappers = layer-level kill switches. ✅
- **H — Human Architecture = Founder Capacity.** The **Third Anchor is the whole risk.** Controls:
  - Treat the founder as a single point of failure; the mitigation is **maximal agent leverage** (the
    Pod Leader's ~50× leverage, here literal) + **ruthless scope discipline** (the spend ladder prevents
    open-ended GPU babysitting).
  - **Bus-factor / continuity:** everything durable lives in the repo/S3, not the founder's head — the
    repo's "restart-resilient" rule is the continuity plan. Keep it that way; it's a real asset in
    diligence.
  - **The Binding Problem, inverted:** there's no team to retain — the retention target is *the founder's
    own sustained engagement*. Engineer Consequence (the work visibly matters for a neglected disease),
    Legibility (the daily status board + gates make progress visible), Identity (honest-science purpose).
- **A — Adaptive Architecture.** Provider-agnostic backend = modular, swappable compute. ✅ Keep the
  pipeline target-agnostic so it retargets to new families without a rewrite (this is what makes the
  Domain Collapse Engine possible).
- **P — Purpose Control.** The MTP protocol above; Identity disqualifiers explicit. ✅
- **E — Ecosystem Trust.** For a shop selling predictions to others: **the moat is the trusted
  accountability stack, not the smartest model.** Any customer engagement needs a codesigned
  data/liability boundary *before* work starts (esp. if handling a partner's proprietary structures) —
  see §5 Six Questions.

**SHAPE score (honest):** ~**16/25**. Strong on Adaptive/Purpose/Safe; the drag is Human Architecture
(single substrate) and formalized Ecosystem Trust for paid work. **Do not dress this up** — the plan's
job is to make the founder-capacity constraint explicit and engineer around it.

same GPUs.** Use `templates/decision-trace-template.md`.

**Cognitive captivity (Design Condition 3):** the firm's *reasoning* runs on one foundation-model

repo, not in a vendor); treat the model as swappable. "Own your loop; swap the model, keep the veteran"

## 6. The business model — how the paper becomes a viable one-person company

ExO economics applied: **tokens-as-COGS** (the only real cost is GPU-$ per predicted candidate matrix —
agent labor is flat-rate/≈$0), **per-outcome pricing**, and a **Value Moat** that is the validated
pipeline + benchmark/trace library, not any single result.

### 6.1 Three revenue engines (run as a portfolio; sequence by SHAPE-safety and non-dilutiveness)

1. **Non-dilutive, mission-aligned funding (PRIMARY, do first).**
   - Rare-sarcoma / EMC foundations, NR4A/nuclear-receptor philanthropic funders, and method-rigor
     grants (SBIR/STTR-style where a solo/µ-entity qualifies; foundation research grants).
   - *Sold thing:* the honest, benchmarked NR4A3-selectivity program + reproducible pipeline as a
     fundable research asset. The preprint is the credibility artifact.
   - *Why first:* founder-owned, integrity-preserving, and it funds the exact GPU ladder already planned.
     Per-outcome framing: grant milestones map 1:1 onto the STRATEGY.md rungs (each GO/NO-GO is a milestone).

2. **Fee-for-service computational campaigns (SCALABLE, do second).**
   - A µ-CRO offering: run the *validated* ternary-cooperativity / paralogue-selectivity pipeline on
     **someone else's** TPD target (the many PROTAC/molecular-glue/degrader startups need exactly this
     and mostly lack an honest in-silico selectivity capability).
   - *Per-outcome pricing:* price per campaign / per delivered selectivity-ranked candidate set, with
     GPU-$ as transparent COGS + a margin for the validated pipeline + the honest-negative guarantee.
   - *Requires:* Val B-full passing (the ternary method must be benchmarked before it's sold — this is
     already the paper's gate, so the science ladder and the commercial ladder are the *same* ladder).
   - *Ecosystem Trust prerequisite:* codesigned data/liability boundary + Six-Questions treatment of any
     client structures **before** work starts (§4-E).

3. **Asset out-licensing (OPTIONAL, longest horizon, honest dependency).**
   - The NR4A3-selective **predicted candidate matrix** as IP to license to a partner who *can* run wet
     validation (foundation, academic lab, or pharma). **Hard, honestly-stated external dependency:** the
     candidates are computationally prioritized, degradation experimentally unvalidated — value is realized
     only when a partner validates. Do not build the plan's viability on this rung.

### 6.2 The commercialization ladder (spend-gated, mirrors the science ladder)

Legend mirrors STRATEGY.md: `[ ]` pending. Each rung has a GO/NO-GO. **No dilutive or irreversible move
before its cheaper gate passes.**

- **Rung A — Credibility anchor ($0 new).** Finish the science ladder through the benchmarks that are
  already planned; **post the preprint** (the MTP/credibility artifact). GO = red-team clean + sign-off.
- **Rung B — Productize the moat ($0).** Package the pipeline as a reproducible, documented,
  target-agnostic artifact (open-core: public method + benchmarks; private = the campaign service &
  trace library). Emit decision traces from every run (LEARN → moat). *Steinberger flywheel:* public
  reproducibility → reputation → inbound.
- **Rung C — Non-dilutive pipeline (founder time only).** Prepare + submit 2–3 foundation/grant
  applications keyed to EMC/NR4A rigor, each milestone-mapped to a science rung. GO = one grant engaged.
  *(Preparation is self-doable now; the outward-facing submit is the human-gated, sign-off step.)*
- **Rung D — First paid campaign (per-outcome).** Only after Val B-full passes: offer one fee-for-service
  ternary-selectivity campaign to a TPD team, priced per outcome with GPU-$ COGS transparent. GO =
  one paid engagement delivered clean with the honest-negative guarantee intact.
- **Rung E — Cascade (Domain Collapse Engine).** Reuse the target-agnostic pipeline on the next target
  family; each campaign's benchmarks/traces compound the moat. Run as a portfolio (most leads won't
  convert — that's expected; premature termination of the *model* is the enemy, not individual misses).

### 6.3 Unit economics (tokens-as-COGS)

- **COGS per campaign ≈ GPU-$** (the STRATEGY.md anchors: ~$18–60 RBFE, ~$40–288 ternary, full program
  ~$0.8–1.4k *only if every gate says GO*). Agent labor ≈ $0 (flat-rate). This is an **extraordinary
  gross-margin structure** for a services business — the ExO "smaller human core, tokens as COGS" shape,
  literally.
- **The CFO test (here, the founder):** *can you state the exact GPU-$ per delivered candidate matrix?*
  — Yes, from realized GPU-h. That's the pricing floor. Anything above it is margin on the validated
  pipeline + honesty guarantee.

## 7. REWRITE — the migration playbook, mapped to the build (Direct Mode, sequence non-negotiable)

1. **BACKCAST & DEFINE** *(this document is the first pass).* Output: signed Destination Architecture
   (§3), MTP protocol, Five Design Conditions as the exit gate. **Founder can run this in a one-day
   workshop** (Tier-2 rule). *Next action:* ratify §3.1 MTP + §6.2 ladder.
2. **ASSESS & PREPARE** *(§2 done).* On-ramp = **90-Day Sprint on the highest-leverage commercial
   workflow** (grant #1 + the GOVERN eval), retake the readiness score each quarter.
3. **EXTRACT** — the highest-value first agent is an **elicitation agent, not a task executor**: extract
   the founder's tacit knowledge (why each gate, what "defensible" means, the disqualifiers) into the MTP
   protocol + GOVERN eval so agents reason from it. Produce a **Workflow Data Manifest** for the first
   commercial workflow (grant prep): every source, why, read/write, sensitivity, owner.
4. **DIAGNOSE & STRIP** — **Task Decomposition Matrix** on the founder's week: score each task
   (science-run / writing / grant-prep / client-comms / governance) 1–5 for agent-readiness. Push
   coordination-heavy, low-judgment work (status assembly, literature fetch, reproducibility packaging,
   draft generation) to agents; reserve founder judgment for the irreversible/high-stakes calls
   (over-claim risk, spend gates, submit/post). **This collapses the decision-layer congestion the
   Tokenmaxxing Latency flag found.**
5. **BUILD & PROVE** — Decision Handover Waves, easiest first, ≤2–3 parallel: (Wave 1) auto-assemble the
   status board + reproducibility artifact; (Wave 2) draft grant/client deliverables under the GOVERN
   eval; (Wave 3) higher-judgment selectivity-narrative drafting. **Parallel-run then deprecate** — the
   agent draft runs alongside the founder's until the **human-override rate falls** (the test of a real
   twin). Cold-start feeds: historical replay (past manuscripts), shadow comparison, override capture.
6. **REWIRE & EVOLVE** — the "org" is founder + agent pods on a thin accountability spine. Make the
   **Self-Disruption Probe** permanent: SENSE feeds (new methods via method-watch, new fundable
   targets, new TPD prospects) continuously spawn candidate campaigns; deprecate dead leads fast.

**GOVERN/ASSURE stays alive across all six steps** — alert-only now (the eval checklist flags
over-claims), escalating to a hard gate on the three irreversible acts.
